## Таблица атрибутов и настроек JSON

| Атрибут / настройка | Что делает | Пример |
|---------------------|------------|--------|
| `[JsonPropertyName("name")]` | Задаёт имя поля в JSON | `"name"` вместо `"Name"` |
| `[JsonIgnore]` | Поле не попадает в JSON | Пароли, внутренние поля |
| `PropertyNamingPolicy.CamelCase` | Все поля в camelCase | `"realName"` вместо `"RealName"` |
| `JsonStringEnumConverter` | Enum как строка | `"Marvel"` вместо `0` |
| `WriteIndented = true` | JSON с отступами | Для удобства при разработке |

## Главные выводы

### 1. Что такое сериализация и десериализация

- **Сериализация** — это процесс превращения C#-объекта в JSON-строку. Сервер отправляет клиенту именно строку.
- **Десериализация** — обратный процесс: JSON-строка превращается обратно в C#-объект.

### 2. Зачем нужен [JsonIgnore] — реальный пример

`[JsonIgnore]` исключает свойство из JSON-ответа. Это нужно, когда поле содержит служебную информацию, не предназначенную для клиента.

### 3. Почему enum лучше сериализовывать строкой, а не числом

- Когда enum сериализуется числом (`"universe": 0`), клиент не понимает, что означает `0` — нужно смотреть документацию или исходный код.
- Когда enum сериализуется строкой (`"universe": "Marvel"`), клиент сразу видит понятное значение.
- Строковый формат менее подвержен ошибкам при добавлении новых значений в enum.

### 4. Что изменилось в ответе после настройки CamelCase

До настройки `CamelCase` поля в JSON выглядели так:
```json
{
    "RealName": "Питер Паркер",
    "PowerLevel": 75
}
```

После настройки CamelCase поля стали с маленькой буквы:
```json
{
    "realName": "Питер Паркер",
    "powerLevel": 75
}
```
